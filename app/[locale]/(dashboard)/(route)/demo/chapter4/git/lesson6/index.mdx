## 一、Git 仓库的四个区域
在开始之前，需要理解 Git 中代码存在的四个位置：

1. 工作目录（Working Directory）：本地编辑文件的区域
2. 暂存区（Staging Area）：临时存放修改内容的地方
3. 本地仓库（Local Repository）：存储已提交修改的地方
4. 远程仓库（Remote Repository）：如 GitHub 这样的远程服务器

## 二、连接步骤
## 1. 初始化本地仓库
```bash
## 在项目目录下初始化 Git 仓库
git init
 ```

## 2. 添加远程仓库
```bash
## 添加远程仓库，使用 SSH 地址
git remote add origin git@github.com:用户名/仓库名.git

## 验证远程仓库配置
git remote -v
 ```

## 3. 准备首次提交
```bash
# 添加所有文件到暂存区
git add .

# 首次提交
git commit -m "Initial commit"
 ```

## 4. 推送到远程仓库
```bash
# 设置上游分支并推送
git push -u origin main
 ```

## 三、常见操作
## 1. 查看远程仓库信息
```bash
# 查看所有远程仓库
git remote

# 查看远程仓库详细信息
git remote -v
 ```

## 2. 修改远程仓库
```bash
# 修改远程仓库地址
git remote set-url origin 新的仓库地址

# 重命名远程仓库
git remote rename origin new_name
 ```

## 3. 删除远程仓库连接
```bash
# 删除远程仓库连接
git remote remove origin
 ```

## 四、注意事项
1. 确保本地分支名称与远程分支一致（现在通常默认是 main ）
2. 首次推送使用 -u 参数设置上游分支，后续可直接使用 git push
3. 如果远程仓库已有内容，建议先拉取再推送：
```bash
git pull origin main --allow-unrelated-histories
 ```

4. 如果遇到推送失败，可能是权限问题或远程仓库有更新，需要先拉取最新代码
## 五、验证连接
```bash
# 测试连接是否成功
git fetch

# 查看远程分支
git branch -r
 ```

这样，你的本地仓库就成功连接到了远程仓库，可以开始进行代码的推送和拉取操作了。
