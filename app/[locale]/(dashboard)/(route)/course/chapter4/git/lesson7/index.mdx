在推送代码之前，需要了解代码在 Git 中的四个存储位置：

1. 工作目录：本地编辑代码的地方
2. 暂存区：临时存放修改的地方
3. 本地仓库：存储已提交修改的地方
4. 远程仓库：GitHub 上的代码存储位置

## 二、推送代码的完整流程
## 1. 查看当前状态
```bash
# 查看哪些文件被修改
git status
 ```

## 2. 添加文件到暂存区
```bash
# 添加单个文件
git add 文件名

# 添加所有修改的文件
git add .

# 添加所有更新的文件（不包括新文件）
git add -u
 ```

## 3. 提交到本地仓库
```bash
# 提交并添加提交信息
git commit -m "提交说明"
 ```

## 4. 推送到远程仓库
```bash
# 首次推送，设置上游分支
git push -u origin main

# 后续推送可以简化为
git push
 ```

## 三、常见场景
## 1. 首次推送项目
```bash
# 初始化仓库
git init

# 添加远程仓库
git remote add origin git@github.com:用户名/仓库名.git

# 添加所有文件
git add .

# 首次提交
git commit -m "Initial commit"

# 推送到远程仓库
git push -u origin main
 ```

## 2. 更新现有项目
```bash
# 查看修改
git status

# 添加修改
git add .

# 提交修改
git commit -m "更新说明"

# 推送到远程
git push
 ```

## 四、注意事项
1. 推送前先拉取最新代码
```bash
git pull
 ```

2. 如果遇到冲突
```bash
# 查看冲突文件
git status

# 解决冲突后重新添加和提交
git add .
git commit -m "解决冲突"
git push
 ```

3. 如果想查看提交历史
```bash
# 查看提交记录
git log

# 查看简化的提交记录
git log --oneline
 ```
