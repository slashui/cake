---
title: 缓存
vid: 70d5e6dd45c271f0bfcc5017f0f80102
---

本节聊聊 Next.js 中服务器组件获取数据的额外优势——缓存。缓存是将数据存储在更快访问的地方（如内存、文件系统或网络），其中网络访问最慢。Next.js 内置数据缓存，调用 fetch 函数获取数据时，自动将结果存储在基于文件系统的数据缓存中。下次访问相同 URL 时，Next.js 直接从缓存读取，避免再次请求后端（如 JSON Placeholder）。可通过 fetch 的选项对象控制缓存行为：1) 设置 `cache: 'no-store'` 禁用缓存，适合频繁变化的数据；2) 设置 `next: { revalidate: 10 }`，每 10 秒后台更新数据，保持数据新鲜。需注意，仅 fetch 函数支持此缓存机制，使用第三方库（如 Axios）无法享受缓存功能。
