本节讲解了在 Next.js 中客户端和服务器端获取数据的区别。客户端获取数据通常使用 React 的 useState 和 useEffect 钩子（或 React Query），但存在问题：bundle 体积随组件增加而增大、客户端资源消耗高、搜索引擎无法索引内容、API 密钥等敏感数据易暴露，且需额外一次后端请求。服务器端获取数据可解决这些问题，在服务器组件中直接使用 fetch 函数（无需状态或效果钩子），通过 async/await 从后端（如 JSON Placeholder 的假 API）获取数据，渲染后发送到客户端，减少 bundle 体积、提升性能、利于搜索引擎索引，并保护敏感数据。示例中在 users/page.tsx 中获取用户列表，定义 TypeScript 接口 User 指定数据结构，确保类型安全和代码补全。服务器组件渲染用户列表，直接在 HTML 文档中返回内容，避免客户端额外请求，优于客户端渲染的空白文档加后续请求模式。推荐优先在服务器组件中获取数据。
