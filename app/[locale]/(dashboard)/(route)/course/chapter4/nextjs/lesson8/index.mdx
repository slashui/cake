本节讲解了 Next.js 中客户端渲染（CSR）和服务器端渲染（SSR）的区别及优缺点。CSR 需将所有组件打包发送到客户端， bundle 体积随应用增长而增大，消耗客户端资源，且搜索引擎爬虫无法查看内容，API 密钥等敏感数据也可能暴露。SSR 在服务器渲染组件，仅发送必要内容，减少 bundle 体积，节省客户端资源，利于搜索引擎索引，并保护敏感数据。但服务器组件（默认在 app 文件夹中）无法处理交互事件、浏览器 API 或状态管理，需用客户端组件实现交互功能。实际应用中，推荐默认使用服务器组件，仅在必要时（如处理点击事件）使用客户端组件。例如，产品列表页面的大部分组件（如导航、侧边栏）可在服务器渲染，仅将交互按钮（如“加入购物车”）提取为客户端组件，通过 use client 指令标记。演示中创建 ProductCard 组件，尝试添加交互按钮导致错误，解决方法是将按钮提取为单独的 AddToCart 客户端组件，保持 ProductCard 在服务器渲染。课程还提到新 App Router 支持服务器组件，旧 Pages Router 不支持，建议使用 App Router。
